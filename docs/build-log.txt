     Quake ANSI-Terminal Renderer ("SpectreDoom")

     Context

     Build a renderer for Quake that outputs to an ANSI terminal, using the
     Unicode half-block
     character ‚ĖÄ (U+2580, fg = top pixel / bg = bottom pixel) to double vertical
     pixel density,
     running at original game speed. The repo
     (/home/user/repos/SpectreDoom) is empty ‚ÄĒ
     this is a greenfield fork.

     User decisions (confirmed):
     - Fork base: TyrQuake (mirror: https://github.com/sezero/tyrquake, v0.71) ‚ÄĒ
     64-bit-clean
     WinQuake software renderer, plain-make build, pluggable driver targets.
     - Terminal support: modern terminals only (kitty, ghostty, wezterm, foot,
     recent alacritty) ‚ÄĒ
     we depend on the kitty keyboard protocol for real key press/release events.
     - Scope: core video + keyboard + sound. No mouse look, no 256-color
     fallback (v1).

     "Original speeds" defined: Quake's engine caps at 72 fps (NQ/host.c:500,
     realtime - oldrealtime < 1.0/72.0). Target: sustain 72 fps at typical
     fullscreen terminal
     sizes; verify with timedemo demo1.

     Why TyrQuake fits (verified in source, cloned at

     /tmp/claude-1000/.../scratchpad/tyrquake)

     - Makefile has pluggable VID_TARGET / IN_TARGET / SND_TARGET (incl. null
     skeletons:
     common/vid_null.c 171 lines, common/in_null.c 72 lines). We add a term
     target for vid+in.
     - Driver contract (include/vid.h): implement
     VID_Init/VID_SetMode/VID_Update(vrect_t*)/ 
     VID_SetPalette/VID_ShiftPalette/VID_Shutdown/VID_ProcessEvents etc.;
     renderer draws 8-bit
     palettized pixels into vid.buffer (viddef_t).
     - Input contract: Key_Event(knum_t key, qboolean down) (include/keys.h:288)
     ‚ÄĒ exactly the
     press/release model the kitty protocol provides. Key codes are SDL1-style
     (K_ESCAPE=27,
     K_UPARROW=273, ...).
     - Resolution limits: MAXWIDTH 2040, MAXHEIGHT 1200 (include/r_shared.h) ‚ÄĒ
     comfortably
     above any terminal grid (grid pixels = cols √ó 2¬∑rows).
     - Sound: existing snd_sdl.c (SND_TARGET=sdl, audio-only SDL); fallbacks
     already in tree:
     pulseaudio, oss, sndio.
     - Reference for a complete driver: common/vid_sdl.c (571 lines).

     Architecture

     Fork lives at repo root, preserving upstream history:
     git remote add upstream https://github.com/sezero/tyrquake.git && git fetch
     upstream,
     point main at upstream/master, commit our work on top (keeps future
     upstream merges possible).

     New files (following existing driver-file conventions):
     - common/term_tty.c + include/term_tty.h ‚ÄĒ shared tty layer used by both
     vid and input:
     termios raw mode, alt-screen/cursor/autowrap setup, kitty-keyboard
     push/pop, capability probe,
     SIGWINCH/SIGTSTP/SIGCONT handling, and crash-safe restore (atexit +
     SIGSEGV/SIGTERM/SIGABRT
     handlers that emit the restore sequence ‚ÄĒ a crashed game must never leave
     the tty raw).
     - common/vid_term.c ‚ÄĒ the renderer backend.
     - common/in_term.c ‚ÄĒ keyboard backend.
     - Makefile: VID_TARGET=term ‚Üí vid_term.o term_tty.o; IN_TARGET=term ‚Üí
     in_term.o;
     plus a convenience target (e.g. make terminal) that sets
     VID_TARGET=term IN_TARGET=term SND_TARGET=sdl and builds bin/tyr-quake.

     Video driver (vid_term.c)

     Mode: one mode derived from TIOCGWINSZ at startup: render W = cols (clamped
     to a
     multiple of 8 ‚ÄĒ check the software renderer's rowbytes/surface-cache
     alignment assumptions,
     vid_sdl.c shows what's required), render H = 2 √ó rows. WinQuake's minimum
     is 320√ó200 (HUD/
     console art is 320 wide), so:
     - Terminal grid ‚Č• 320√ó200 ‚Üí render at native grid size, 1 pixel per
     half-cell.
     - Smaller terminal ‚Üí render at 320√ó200 and nearest-neighbor downscale to
     the cell grid via
     precomputed x/y index maps (cheap; documented min recommended ~160√ó50 chars
     for readability).

     Palette: VID_SetPalette builds LUTs from the 256-entry palette: packed RGB
     per index plus
     precomputed SGR fragments per index ("38;2;R;G;B" / "48;2;R;G;B" strings)
     so the hot loop
     never formats integers. VID_ShiftPalette (pain/underwater flashes) rebuilds
     LUTs and marks the
     whole grid dirty.

     VID_Update hot path:
     1. Sample framebuffer into a cell grid: each cell = (top_index, 
     bottom_index) packed in 16 bits.
     2. Diff against previous frame's grid. For changed cells emit: cursor move
     (CSI row;colH) only
     when skipping, SGR only when fg/bg palette index actually changes
     (run-length elision),
     glyph ‚ĖÄ (3 bytes UTF-8); when top == bottom, emit a space with bg color
     instead (1 byte,
     fg state untouched).
     3. Wrap the frame in synchronized-update guards (CSI ?2026h ‚Ä¶ CSI ?2026l) ‚ÄĒ
     no tearing.
     4. Single write() of the assembled buffer to the tty.
     5. Backpressure/frame-drop: if the previous frame's write time exceeded the
     1/72 s budget,
     skip emitting (game simulation continues at full speed; terminal never
     throttles the engine).

     Resize: SIGWINCH sets a flag; next frame re-queries size and re-runs
     VID_SetMode
     (realloc buffers, full redraw, vid.recalc_refdef = 1).

     Console/stdout conflict: Sys_Printf in common/sys_unix.c writes to stdout,
     which would
     corrupt the display. While terminal video is active, redirect engine
     console output to a log file
     (flush real errors to stderr only after terminal restore on exit/crash).

     Input driver (in_term.c)

     - Probe kitty keyboard protocol at init (CSI ? u query); if unsupported,
     exit with a clear
     message naming supported terminals (also note: not under tmux for v1).
     - Push progressive enhancement flags (disambiguate escape codes + report
     event types + report
     alternate keys); pop on shutdown.
     - CSI-u state-machine parser in VID_ProcessEvents/Sys_SendKeyEvents: event
     type 1=press ‚Üí
     Key_Event(k, true), 2=repeat ‚Üí also down=true (console typing needs
     repeats; game key
     state is unaffected), 3=release ‚Üí Key_Event(k, false). Map kitty
     functional-key codes
     (arrows, F-keys, modifiers) to TyrQuake knum_t.
     - Raw mode disables ISIG; keep an emergency quit (e.g. Ctrl+C thrice ‚Üí
     clean shutdown) in
     addition to Quake's own quit menu.

     Sound

     SND_TARGET=sdl using existing common/snd_sdl.c with SDL audio subsystem
     only. Early check:
     confirm sdl_common.c init doesn't drag in SDL video; if it does, patch to
     SDL_InitSubSystem(SDL_INIT_AUDIO) or fall back to the existing pulseaudio
     target.

     Game data

     Engine loads id1/pak0.pak from -basedir. Add scripts/fetch-shareware.sh to
     download the
     redistributable Quake 1.06 shareware pak for testing (not committed to the
     repo). Registered
     paks work if the user supplies them.

     Implementation order

     1. Baseline: import upstream into the repo; build stock VID_TARGET=x11 (or
     sdl) and run
     with shareware data ‚ÄĒ prove the engine works before touching it.
     2. Makefile term targets + term_tty.c (raw mode, alt screen,
     restore-on-crash) + stub vid/in
     that builds and runs headless.
     3. vid_term.c naive full-frame renderer (no diff) ‚ÄĒ get a picture.
     4. Downscale path + SIGWINCH resize.
     5. Optimize: cell diff, SGR run elision, precomputed color strings,
     sync-update wrap,
     frame-drop policy. Measure with timedemo demo1.
     6. in_term.c kitty protocol: full key map, press/release play test.
     7. Sound build + test; console log redirection; suspend/resume
     (SIGTSTP/SIGCONT) handling.
     8. Polish: README (requirements, terminal recommendations, min size),
     fetch-shareware script.

     Verification

     - Speed: +timedemo demo1 (uncapped) must report well above 72 fps at a
     fullscreen
     kitty/ghostty grid (~240√ó67 cells); normal play then holds the engine's 72
     fps cap.
     host_speeds 1 to confirm where time goes if not.
     - Feel: manual play ‚ÄĒ WASD/arrow movement starts and stops instantly
     (release events),
     console typing with key repeat works, pain-flash palette shift renders,
     mid-game resize
     reflows, Escape menu navigable.
     - Robustness: quit, Ctrl+Z/fg, and kill -SEGV all leave the terminal usable
     (cooked mode,
     main screen, cursor visible).
     - Sound: in-game audio plays with no video-subsystem dependency.

     Known risks

     - Software renderer width-alignment assumptions (multiple-of-8) ‚ÄĒ verify
     against vid_sdl.c
     before choosing exact grid clamping.
     - snd_sdl possibly requiring SDL video init ‚ÄĒ checked early (step 7 has
     fallbacks in-tree).
     - tmux/screen don't pass the kitty protocol through reliably ‚ÄĒ documented
     as unsupported in v1.
     - Tiny terminals (80√ó24) will render but be borderline unplayable ‚ÄĒ
     documented minimum instead
     of engineering around it.

  What was built

  SpectreDoom is a TyrQuake fork (full upstream history preserved for future
  merges) with three new files plus small patches:

  - common/vid_term.c ‚ÄĒ samples the 8-bit software framebuffer into a per-cell
  (top, bottom) pixel pair, renders each cell as ‚ĖÄ (U+2580) with truecolor
  foreground/background, diffs against the previous frame, elides redundant
  SGR/cursor sequences, and wraps frames in synchronized-update guards so
  nothing tears. If the tty falls behind, at most every other frame is dropped
  so the engine never slows down.
  - common/in_term.c ‚ÄĒ kitty keyboard protocol parser giving real key
  press/release events (the thing that makes WASD movement possible in a
  terminal), with repeat filtering and a triple-Ctrl+C emergency quit.
  - common/term_tty.c ‚ÄĒ raw mode, alt screen, protocol probing, stdout diverted
  to console.log, and async-signal-safe terminal restore on every exit path.
  - Patches: two upstream bug fixes (-nosound segfault in S_ClearOverflow;
  Sys_Error now restores the display before printing), Makefile term driver
  targets, make terminal, README, and scripts/fetch-shareware.sh.

  Verified (via a Python pty harness, since this session has no real terminal)

  - Speed: timedemo demo1 at a typical fullscreen grid (240√ó67 cells) runs 332 
  fps ‚ÄĒ 4.6√ó the engine's native 72 fps cap, so normal play holds original speed
  with big headroom. Even an extreme 800√ó200 grid sustains ~72 fps.
  - Correctness: decoded captured ANSI streams back into images ‚ÄĒ demo playback,
  HUD, menu, particles, and the 80√ó24 downscale path all render correctly.
  - Input: scripted kitty-protocol events opened the menu with Escape and moved
  the cursor with arrow keys, confirming press/release parsing end-to-end.
  - Robustness: SIGTERM and SIGSEGV both leave the terminal restored (verified
  byte-for-byte in the capture tails); sound initializes with no SDL video
  dependency.
